Git提交本地代码到GitHub仓库

Posted on 2024-05-05 In Git笔记
1.首次提交

① 初始化命令

通过命令 git init 把这个目录变成 git 可以管理的仓库

② 将文件添加到暂存区

通过命令 git add . 将所有文件添加到暂存区，注意后面的小数点“.”，意思是添加当前目录下的所有文件，如果需添加指定文件或文件夹，可以在 git add 命令后面直接加上其名称

③ 把文件提交到本地仓库

通过命令 git commit 将文件提交到本地仓库，引号用来描述你本次提交的代码都做了哪些操作

④ 关联远程仓库 github 或 gitee

通过命令 git remote add origin 关联远程库，add 后面加上远程仓库的地址

⑤ 本地库内容推送到远程

通过命令 git pull 同步并合并本地仓库到远程仓库

使用 git push 命令，实际上是把当前分支 main 推送到远程仓库。执行此命令后会要求输入用户名、密码，验证通过后即开始上传，如果在之前本地初始化 git 设置了名称或者邮箱，这一步就不会要求输入，会直接上传。

⭐ 如遇报错 error:error: src refspec main does not match any/ error: failed to push some refs to

  • 错误原因：Github 工程默认名从 master 变成了 main
